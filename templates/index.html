Система реализации АПИ (АПИратор)

Данная система реализована на основе микрофреймворка flask. Систему предпологается использовать в качестве каркаса для веб проектов.

АПИратор - Является небольшой оберткой над проектом flask. Реализованной для личного использования и удобства. Разработка новых модулей ведется в следующем порядке:

  • Создание класса;
  • Методы АПИ;
  • Регистрация класса.

Создание класса - Класс модуля логически включает в себя реализацию какого-либо процесса. Данный клас должен быть наследником базового класса ObjectAPI. Реализация классов подобного рода в структуре проекта, распологается в папке LibAPI.

Методы АПИ - Создоваемы класс может содержать методы, использование которых возможно по ссылке, такой метод создается с префиксом "api_". Например, метод:
class Article(ObjectAPI):
---- def api_get_art(self):
-------- . . .
будет доступен по ссылке http://ваш домен/Article/get_art. Функция может возвращать любой результат который позволяет протакол http.

Регистрация класса - Для регистрации модуля и его активации требуется проимпортировать его в файл __init__.py пакета LibAPI.
